API 使用手冊

從建立帳號、取得 API Key，到送出第一個 OpenAI 相容的推論請求。

開始使用 註冊與登入 取得 API Key 呼叫推論 API 錯誤處理

開始使用

Relay LLM 提供 OpenAI 風格的 Chat Completions 介面。MVP 目前使用 echo 回應，會將最後一則 user 訊息包含在結果內，不會呼叫真實模型。

  1. 註冊帳號並登入。
  2. 在控制台建立 API Key。
  3. 以 Bearer Auth 呼叫推論端點。

完整 API Key 只在建立時顯示一次。請放在伺服器端的環境變數，不要提交到版本控制或放進瀏覽器程式碼。

註冊與登入

POST /api/register

使用者名稱需為 3-64 字元，可使用英文字母、數字、底線、連字號或句點；密碼需為 8-128 字元。

curl -X POST http://127.0.0.1:8000/api/register \
  -H 'Content-Type: application/json' \
  -d '{"username":"alice","password":"correct-horse-battery-staple"}'

POST /api/login

登入成功會透過 Set-Cookie 建立 Session。curl 可用 -c 將 Cookie 存到檔案。

curl -i -c cookies.txt \
  -X POST http://127.0.0.1:8000/api/login \
  -H 'Content-Type: application/json' \
  -d '{"username":"alice","password":"correct-horse-battery-staple"}'

取得 API Key

POST /api/keys

帶入登入時保存的 Session Cookie。未登入會收到 401。

curl -X POST http://127.0.0.1:8000/api/keys \
  -b cookies.txt

成功回應中的 message.api_key 即為完整金鑰：

{
  "success": true,
  "message": {
    "api_key": "llm_live_REPLACE_ME"
  }
}

呼叫推論 API

POST /v1/chat/completions

將 API Key 放在 Authorization header。請求需包含模型名稱與訊息陣列。

curl -X POST http://127.0.0.1:8000/v1/chat/completions \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer llm_live_REPLACE_ME' \
  -d '{
    "model": "mock-echo-1",
    "messages": [
      {"role": "system", "content": "Answer briefly."},
      {"role": "user", "content": "Hello from Relay"}
    ]
  }'

成功回應保留 OpenAI 相容的頂層 choices 與 usage。Token 數為一致的近似值，且 total_tokens 永遠等於 prompt 與 completion 之和。

{
  "success": true,
  "message": {},
  "id": "chatcmpl_...",
  "object": "chat.completion",
  "created": 1750000000,
  "model": "mock-echo-1",
  "choices": [{
    "index": 0,
    "message": {
      "role": "assistant",
      "content": "Echo: Hello from Relay"
    },
    "finish_reason": "stop"
  }],
  "usage": {
    "prompt_tokens": 8,
    "completion_tokens": 6,
    "total_tokens": 14
  }
}

錯誤處理

所有 API 錯誤都是 JSON，不會回傳 HTML 或 stack trace。可依 HTTP 狀態碼與 message.code 判斷原因。

{
  "success": false,
  "message": {
    "code": "invalid_api_key",
    "detail": "A valid Bearer API key is required."
  }
}

400 表示輸入格式不正確，401 表示 Session 或 API Key 無效，409 表示使用者名稱已存在。
